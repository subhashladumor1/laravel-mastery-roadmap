## Guidelines

1.  **Format**: All code examples must be PSR-12 compliant (use Laravel Pint).
2.  **Structure**:
    *   **Concept**: Explanation.
    *   **Code**: Real-world example.
    *   **When to Use**: Valid use-case.
    *   **When NOT to Use**: Anti-pattern.
    *   **Interview Insight**: A "Senior Level" answer.
3.  **No Fluff**: Get to the point.
4.  **No `foo`, `bar`, `baz`**: Use real domain models (`User`, `Invoice`, `Subscription`).

## Pull Request Process

1.  Fork the repo.
2.  Create a branch: `feature/add-redis-tip`.
3.  Add your `.md` file in the correct directory.
4.  Run `npm run lint` or `vendor/bin/pint`.
5.  Submit PR with a "Why this matters" description.
